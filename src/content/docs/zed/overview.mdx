## Was ist ZED?

ZED ist ein moderner, hochperformanter Code-Editor (Rust-basiert), entwickelt von Zed Industries. Er ist kostenlos, Open Source (GPL) und auf Linux, macOS und Windows verfügbar.

Download: [zed.dev/download](https://zed.dev/download)

## Warum ZED für Collective Context?

ZED ist die **Referenz-Implementierung für ACP** (Agent Client Protocol). Das bedeutet:

- Alle ACP-Agenten werden primär für ZED entwickelt und getestet
- ZED hat das umfangreichste ACP-Ökosystem (mehr Agenten als jeder andere Editor)
- Das Agent Panel ist tief in den Editor-Workflow integriert (keine Browser-Tabs, keine Copy-Paste)

## Das Agent Panel

Das Agent Panel ist der zentrale Ort für alle AI-Interaktionen in ZED:

| Shortcut | Funktion |
|----------|----------|
| `Ctrl+?` | Agent Panel öffnen/schließen |
| `+` | Neuen Agenten hinzufügen (aus ACP Registry) |
| `/login` | Agent-Authentifizierung |
| `@Datei` | Datei explizit in Kontext ziehen |

**Was das Agent Panel bietet:**
- Mehrere Agenten gleichzeitig aktiv (jeder in eigenem Tab)
- Inline Diffs mit accept/reject per Hunk
- CLAUDE.md wird automatisch geladen (kein manueller Upload)
- Tool-Calls sichtbar und nachvollziehbar

## Claude Code Web vs. ZED Claude Code Tab

<Tabs>
  <TabItem label="ZED Claude Code Tab">
    **Empfohlen für:** Tägliche Entwicklung, Multi-File-Tasks, Inline-Diffs

    | Feature | Wert |
    |---------|------|
    | Modell | Sonnet 4.6 (Claude Max) |
    | Kosten | Flat Rate (Max-Abo deckt alles ab) |
    | Filesystem-Zugriff | ✅ Direkt |
    | Inline Diffs | ✅ Im Editor (accept/reject per Hunk) |
    | `@file` Kontext | ✅ Ja |
    | Playwright lokal | ✅ Ja |
    | Ollama (indirekt) | ✅ Ja via Script |
    | Session History | ❌ Pro Session |
    | CLAUDE.md auto-load | ✅ Ja |
  </TabItem>
  <TabItem label="Claude Code Web">
    **Empfohlen für:** Planung, Recherche, Sessions ohne lokale Dateien

    | Feature | Wert |
    |---------|------|
    | Modell | Sonnet 4.6 (Claude Max) |
    | Kosten | Flat Rate (Max-Abo deckt alles ab) |
    | Filesystem-Zugriff | ❌ Copy-Paste |
    | Inline Diffs | Im Browser |
    | `@file` Kontext | ❌ Nein |
    | Playwright lokal | ❌ Nein |
    | Ollama (indirekt) | ❌ Nein |
    | Session History | ✅ Ja (Sidebar) |
    | CLAUDE.md auto-load | ✅ Ja |
  </TabItem>
  <TabItem label="Unterschiede">
    Beide Oberflächen nutzen **dasselbe Modell** (Sonnet 4.6) und dasselbe **Max-Abo** (Flat Rate).

    Der entscheidende Unterschied:

    - **ZED Tab** = Agent hat direkten Filesystem-Zugriff → liest, schreibt, committet autonom
    - **Claude Code Web** = Agent arbeitet mit copy-paste → du kontrollierst jeden Transfer

    Für kollaborative AI-Workflows (Dual-Agent, Multi-Repo) ist der **ZED Tab** klar überlegen.
    Für strategische Planung und Recherche ist **Claude Code Web** völlig ausreichend.

    <Aside type="tip">
    `/rename Neuer Titel` in Claude Code (Terminal oder Web) schreibt den Titel ins JSONL und
    synchronisiert ihn mit ZED History und der TUI — alle drei Systeme in Sync.
    </Aside>
  </TabItem>
</Tabs>

</Aside>
